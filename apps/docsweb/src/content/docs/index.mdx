Runmark keeps tracked request intent in `runmark/`, runtime evidence in
`runmark/artifacts/`, and exposes the same execution model through both the CLI
and MCP. Start with the product path below: understand the model, get a demo
project green, inspect a paused run, then move into team and reference docs as
needed.
