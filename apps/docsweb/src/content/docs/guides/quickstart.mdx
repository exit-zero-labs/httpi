`httpi` is a file-based HTTP workflow runner for repositories. It keeps tracked
request intent in `httpi/`, Git-ignored runtime state in `httpi/artifacts/`,
and exposes the same execution model through both a CLI and an MCP server.

> **Transition note:** Runmark is the new product name for `httpi`. The current
> npm package, CLI command, repository, and on-disk project folder still use
> `httpi` while the rename work is staged.

## Install

One package, two surfaces:

```bash
npm install -g @exit-zero-labs/httpi
```

- `httpi` — the CLI for humans and scripts
- `httpi mcp` — the stdio MCP server for coding agents

## Start a project

```bash
mkdir demo-api && cd demo-api
httpi init
```

That gives you a small but real starting point: one environment, one request,
one run, and schema-aware YAML files.

After `init`, the normal first loop is:

```bash
# edit httpi/env/dev.env.yaml so baseUrl points at your service
httpi validate
httpi describe --run smoke
httpi run --run smoke
```

If the flow needs local secrets, add them under `httpi/artifacts/secrets.yaml`:

```yaml
devPassword: swordfish
apiToken: sk_test_123
```

Tracked files can reference `{{secrets.alias}}` or `$ENV:NAME`, but secret
literals should stay out of `httpi/`.

## Project layout

```text
demo-api/
└── httpi/
    ├── config.yaml
    ├── env/
    │   └── dev.env.yaml
    ├── requests/
    │   └── ping.request.yaml
    ├── runs/
    │   └── smoke.run.yaml
    └── artifacts/
        ├── secrets.yaml
        ├── sessions/
        └── history/
```

| Path | Purpose |
| --- | --- |
| `httpi/` | Git-tracked requests, runs, envs, blocks, body templates, and the Git-ignored runtime subtree |
| `httpi/artifacts/` | Local secrets, session state, locks, and captured request artifacts |

## Core commands

| Goal | Command |
| --- | --- |
| scaffold a project | `httpi init` |
| discover requests, runs, envs, or sessions | `httpi list` |
| validate before execution | `httpi validate` |
| inspect a request or run shape | `httpi describe` |
| inspect resolved values and provenance | `httpi explain variables` |
| execute a request or run | `httpi run` |
| inspect a paused or failed session | `httpi session show` |
| inspect captured evidence | `httpi artifacts list` / `httpi artifacts read` |
| continue a paused or failed workflow | `httpi resume` |
